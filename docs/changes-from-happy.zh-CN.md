# Happy → Happy Next 变更记录

[🇬🇧 English](changes-from-happy.md)

本文档总结了 Happy Next 相对于原版 Happy 的变更。

## 概览

| 领域 | 变更内容 |

| 多 Agent | Claude Code、Codex 和 Gemini 均为一等公民 |
| 语音 | 基于 LiveKit 的语音网关，可插拔 STT/LLM/TTS 供应商，并支持朗读（TTS）AI 回复 |
| 工作区 | 多仓库工作树创建、切换、归档和 PR 流程 |
| 代码浏览器 | 文件浏览器、Monaco 编辑器、提交历史、git 暂存/提交/丢弃、图片预览 |
| 会话共享 | 直接邀请和公开链接分享，端到端加密，访问权限控制 |

| Bug 修复 | 250+ 修复，覆盖消息发送、会话、渲染、导航、安全 |

## 编排器

多 Agent 编排系统，定义任务依赖图并自动执行。

- **DAG 任务调度**：定义带依赖关系的任务，Happy 自动解析执行顺序并跨 Agent 调度
- **按任务模型和工作目录**：每个任务可指定特定模型和目录
- **自动审批标志**：配置编排任务的自动审批
- **会话恢复跟进**：通过会话恢复向已完成任务发送后续消息
- **可用模型 API**：`get_context` 暴露每个供应商的可用模型
- **实时监控**：活动徽章显示运行中任务数，状态颜色进度条
- **完整 App UI**：运行列表带过滤标签和运行计数、运行详情页、任务详情页
- **级联取消**：取消运行时级联 `dependency_failed` 到依赖任务
- **MCP 工具集成**：编排器工具注册为 MCP 工具，自动填充工作目录
- **工具描述重写**：编排器重写工具描述以提升 Agent 理解
- **完整 i18n**：所有编排器 UI 完全国际化

## 待发消息队列

CLI 繁忙时发送的消息现在会自动排队并投递。

- **服务端待发队列**：消息按会话在服务端排队
- **自动分发**：CLI 就绪时自动分发排队消息
- **队列面板 UI**：从应用中查看和管理待发消息
- **图片数量徽章**：待发消息预览显示图片附件数量
- **立即发送**：绕过队列立即发送
- **重连同步**：WebSocket 重连时同步队列状态
- **并发安全**：强化分发并发和清理语义

## 多 Agent 支持

原版 Happy 只支持 Claude Code。Happy Next 将 Claude Code、Codex 和 Gemini 视为平等的一等公民。

- **多 Agent 历史页面**，按供应商分标签页（Claude / Codex / Gemini）
- **会话恢复和复制/分叉**，支持全部三个 Agent
- **`/duplicate` 斜杠命令**：打开消息选择器，从对话中任意位置分叉会话，创建一个包含截至选定消息历史的新会话
- **按 Agent 模型选择**，独立缓存，显示上下文窗口
- **费用追踪**，Claude 模型的精确 token 用量（缓存 token、推理 token）
- **Codex 推理强度**配置（low / medium / high / xhigh）
- **ACP（Agent Client Protocol）后端**，替代 MCP 客户端方案
- **Codex App-Server 后端**：替代方案，使用 `codex app-server` JSON-RPC 协议，通过 stdin/stdout 通信，改进会话管理和可靠性
- **Gemini 会话持久化**，JSONL 存储
- **按供应商斜杠命令**：`/clear` 全部 Agent 可用，`/compact` 仅 Claude
- **模型/模式切换**，按会话，实时元数据同步
- **Codex/Gemini diff 处理**：按文件 +N/-N 统计显示在应用中
- **消息回填**，Codex 和 Gemini 恢复/复制会话时可用
- **ACP 结果格式规范化**：Gemini ACP 结果规范化以匹配 Codex 结构
- **工具 ID 前缀回退**：不同前缀的工具 ID 回退匹配
- **Codex v2 协议修复**：解决 v2 协议字段级不兼容
- **动态权限模式**：活跃会话中通过 RPC 变更权限模式
- **Codex 上下文恢复**：`/duplicate` 使用 `thread/resume` 恢复上下文
- **工具名规范化**：`normalizeToolName` 对齐 MCP 工具名与 Codex 约定

## 语音助手（Happy Voice）

Happy Next 包含完整的语音网关栈，从简单的 ElevenLabs 单供应商到完整的 LiveKit 流水线。

- **基于 LiveKit 的语音网关**（`happy-voice`），可插拔供应商
- **按前缀自动切换供应商**：`openai/gpt-4.1-mini` 用于 LLM，`cartesia/sonic-3:voiceId` 用于 TTS，`assemblyai/universal-streaming:en` 用于 STT
- **麦克风静音**，支持所有语音供应商（ElevenLabs、LiveKit/Happy Voice）
- **语音工具**：会话管理（导航、结束对话）、消息发送
- **语音消息发送确认**，可配置倒计时
- **"思考中"指示器**，显示在语音状态栏
- **上下文感知语音**：完整应用状态（会话、git 状态等）作为结构化上下文注入
- **Silero VAD 灵敏度**，通过环境变量调节
- **语音片段合并**，处理被中断的用户话语
- **可配置欢迎消息**，从应用设置中配置
- **系统提示工程**：英文提示、语义 XML 分隔、内联 LLM 提示
- **朗读 AI 回复**：新增一次性 `POST /v1/voice/tts` 接口,合成消息文本(复用已配置的 `AGENT_TTS` 供应商),应用从消息底部的语音按钮播放,同一时刻仅播放一条

## 多仓库工作树工作区

重大新功能：将多个仓库作为统一工作区管理。

- **创建工作区**，从新建会话向导中选择多个仓库
- **RepoPickerBar** 和 **RepoSelector** 组件，用于工作区创建和切换
- **按仓库设置**：分支选择（本地 + 远程）、脚本、配置
- **Git 状态聚合**，跨所有工作区仓库
- **自动生成工作区 `CLAUDE.md` 和 `AGENTS.md`**，含 `@import` 引用
- **工作区生命周期管理**：元数据追踪、git 操作、通过守护进程 RPC 归档/清理
- **工作树合并和 PR 创建**，支持目标分支选择
- **AI 驱动的 PR 代码审查**：一键启动 AI 会话审查 PR，结果发布为 GitHub PR 评论
- **路径显示**，使用 `~/` 符号代替绝对路径

## 代码浏览器和 Git 管理

应用现在包含完整的代码浏览和 git 管理体验。

- **文件浏览器**，目录导航和搜索（当前目录过滤 + 项目全局）
- **文件查看器**，Monaco 编辑器（只读模式查看、编辑模式修改）
- **提交历史**，分支选择器（本地和远程分支）
- **提交详情页**，diff 查看和操作按钮（复制哈希、复制消息）
- **Git 变更页面**：暂存、取消暂存、提交和丢弃变更
- **按文件 diff 统计**（+N/-N 行），支持 Claude Code、Codex 和 Gemini 会话
- **可点击文件路径链接**，Markdown 中支持编辑器定位
- **暂存文件 diff 显示**，精确行数统计
- **Base64 解码**修复 UTF-8（CJK 字符）
- **图片预览**，文件查看器中直接查看图片，支持分享

## 会话共享

与他人通过直接邀请或公开链接分享 AI 编码会话，支持完整的端到端加密。

- **直接分享**，使用 NaCl Box 端到端加密，通过上传内容公钥实现
- **公开链接分享**，使用 token 派生密钥加密
- **访问级别**：查看、编辑和管理权限，服务端强制执行
- **实时同步**：消息、git 状态和语音聊天通过 socket 事件广播给所有共享用户
- **"全部/共享给我/我分享的"过滤标签**，会话列表中
- **共享指示器**，显示在已共享的会话上
- **分享者头像**和**发送者名称**，在共享会话中显示
- **公开分享网页查看器**，无需安装应用即可通过链接访问
- **好友个人主页显示共享会话**
- **权限感知 UI**：输入栏、语音按钮和会话操作根据访问级别自适应
- **服务端访问控制**模块，对消息、RPC 调用和语音进行权限验证
- **公开分享访问日志**

## OpenClaw 集成

通过网关系统连接外部 AI 机器，拥有独立的聊天界面。

- **机器管理**：从应用中添加、编辑和移除 OpenClaw 机器
- **两种连接模式**：Happy 中继（通过 Happy 服务器隧道）或直连 WebSocket 网关
- **Ed25519 密钥交换**，安全的机器配对
- **聊天界面**，实时流式 AI 回复、消息重试和输入指示器
- **会话管理**：创建、浏览和恢复 OpenClaw 会话
- **服务端 CRUD API**，加密元数据和乐观并发控制
- **CLI 隧道管理器**，用于中继连接

## AI 后端配置文件

通过环境变量配置文件为 Claude Code 配置替代 LLM 后端。

- **内置供应商预设**：Anthropic（默认）、DeepSeek、Z.AI、OpenAI、Azure OpenAI、Google AI
- **环境变量映射**：自动映射供应商特定的环境变量（如 `DEEPSEEK_*` → `ANTHROPIC_*`）
- **自定义配置文件**：使用任意环境变量和 `${VAR:-default}` 展开语法创建配置
- **按配置设置**：tmux 会话名称、启动脚本、默认权限模式、Agent 类型兼容性
- **配置编辑器**：全页设置 UI，用于创建和编辑配置文件

- **任务列表页面**，过滤（项目、状态、优先级）、搜索和分页

- **聊天功能**：表情回应、语音消息回放、图片/视频消息、文件卡片
- **乐观 UI**，消息发送支持 HTTP/WebSocket 竞争
- **一键 AI 会话启动**，从任务详情（MCP 服务透传）
- **外部上下文关联**：从 DooTask 启动的会话显示上下文横幅并链接回任务
- **DooTask 连接页面**，登录、验证码支持和字段缓存
- **任务状态管理**：可点击状态徽章，支持工作流转换
- **DooTask 标签页**，主导航中集成，连接账户管理
- **创建任务和项目**，在应用内直接创建，独立表单页面
- **跨平台日期选择器**（`react-native-ui-datepicker`），底部弹窗确认
- **表单缓存**，跨页面导航保持创建表单数据
- **全局化 WebSocket 连接**：单一持久连接，实时任务更新
- **会话关联任务**：会话详情页显示关联的 DooTask 任务
- **持久化连接**：DooTask 连接通过 UserKVStore 保存到服务端
- **简单状态徽章**：无工作流的任务显示简单状态徽章

## 自托管

Happy Next 添加了一等自托管路径。

- **根目录 `docker-compose.yml`**，包含所有服务：Web 应用（Nginx）、API 服务器、语音网关、Postgres、Redis、MinIO
- **独立源架构**：Web、API 和 Voice 各使用不同端口/域名（无路径反向代理）
- **`.env.example`** 作为所有配置的唯一参考
- **运行时环境变量注入**，在 Dockerfile/入口点中用于 Docker 构建
- **`APP_URL`** 配置，用于连接流程
- **`VOICE_TOOL_BRIDGE_BASE_URL`**，用于 Docker 网络中语音到服务器的通信
- **自托管文档**：`docs/self-host.zh-CN.md`

## 同步和消息可靠性

实时同步层的重大可靠性改进。

- **v3 消息 API**，基于 seq 的同步、批量写入和游标分页
- **HTTP 发件箱**，WebSocket 不可用时的可靠消息投递
- **服务端确认消息发送**，失败时重试
- **修复**：首次推送游标跳过、发件箱并发刷新竞争、消息重复、seq 间隙消息丢失、同步游标重置、关闭时发件箱排空

- **消息接收追踪**：CLI 确认消息接收，兼容旧版本
- **happy-wire** 共享协议类型包，去重 CLI/app/server 中的 schema

## CLI 改进

CLI（`happy-next-cli`）收到了大量升级。

- **多 Agent 支持**：Claude Code、Codex（通过 ACP 后端）和 Gemini 作为一等公民
- **会话恢复/复制**，所有 Agent 支持正确的消息回填
- **多仓库工作树**，通过守护进程 RPC 创建/清理工作区
- **Diff 处理**：所有 Agent 的按文件 +N/-N 统计
- **载荷优化**：精简冗余字段，按需延迟加载 diff
- **MCP 配置集中化**，按 Agent 适配器模式（Claude HTTP、Codex stdio、Gemini HTTP）
- **工作树检测**，使用原生 git 而非硬编码路径匹配
- **精确费用计算**，Claude 模型
- **Shell 命令注入修复**，统一转义
- **设置持久化**："不再询问" 工具审批保存到 `settings.local.json`
- **会话标题管理**：`change_title` 工具，支持锁定
- **CI**：冒烟测试，happy-wire 构建依赖
- **`happy update`** 自更新命令
- **`happy --version`** 显示 Claude、Codex 和 Gemini CLI 版本
- **工作树子目录检测**，工作区根目录中
- **最新 CLI 版本**，从 npm 获取而非硬编码最低版本
- **守护进程开机自启动**：`happy daemon enable` / `happy daemon disable`
- **守护进程重启命令**：无需手动 kill 即可重启守护进程
- **Codex v0.116.0 + fast mode**：升级 Codex 并支持快速模式
- **归因设置**：新设置控制提交归因，默认关闭
- **统一系统提示注入**：Codex 和 Gemini 共享提示注入
- **编排器引导**：首轮提示包含编排器使用指南
